# 11. 常见问题

## Q1: 登录后页面空白或 404

**原因**: 动态路由未正确加载。

**解决方案**:

1. 检查后端 `/admin/profile` 接口是否正常返回 `menus` 数据
2. 检查浏览器 SessionStorage 中 `rbac_admin_menus` 是否有值
3. 清除浏览器缓存后重新登录

## Q2: JWT Token 验证失败

**原因**: JWT\_SECRET 配置问题。

**解决方案**:

1. 确认 `.env` 中 `JWT_SECRET` 已设置且长度 ≥ 32 字符
2. 运行 `php setup_jwt_secret.php` 重新生成
3. 确保 `.env` 文件未被 Git 跟踪

## Q3: 接口返回 403 无权限

**原因**: 接口权限未配置或未分配给角色。

**解决方案**:

1. 检查 `sys_api` 表中是否配置了该接口
2. 检查用户角色是否关联了该接口（`sys_role_api`）
3. 超级管理员自动拥有所有权限

## Q4: 登录失败账户锁定

**原因**: 连续登录失败超过阈值（默认5次）。

**解决方案**:

1. 等待锁定时间结束（默认15分钟）
2. 修改 `.env` 中 `AUTH_LOGIN_LOCK_DURATION=0` 禁用锁定
3. 删除 `runtime/cache/` 下对应的缓存文件

## Q5: 前端代理 404

**原因**: Vite 代理配置与后端路由不匹配。

**解决方案**:

1. 检查 `.env.local` 中 `VITE_APP_BASE_API` 和 `VITE_APP_API_BASE_URL`
2. 确保后端服务已启动
3. 检查 Vite 代理配置（`vite.config.ts`）

## Q6: 数据库迁移失败

**原因**: 数据库连接配置或权限问题。

**解决方案**:

1. 确认 MySQL 服务已启动
2. 检查 `.env` 中数据库配置
3. 确认数据库用户有 CREATE/DROP 权限
4. 手动执行 SQL 文件：`mysql -u root -p rbac_system < database/migrations/001_init_schema.sql`

## Q7: 操作日志不记录

**原因**: 事件监听未注册或中间件未生效。

**解决方案**:

1. 检查 `app/event.php` 中 `OperateLog` 事件是否注册
2. 确认路由组已应用 `RecordOperate` 中间件
3. 检查 `app/admin/event/OperateLogEvent.php` 是否存在

## Q8: 前端构建报 TypeScript 错误

**解决方案**:

```bash
# 跳过类型检查构建
npx vite build

# 或修复类型错误后构建
npm run build
```
