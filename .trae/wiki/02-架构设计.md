# 2. 架构设计

## 2.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                     客户端（浏览器）                       │
│  Vue 3 + Ant Design Vue + Pinia + Vue Router             │

                           │ HTTP / HTTPS

┌──────────────────────────▼──────────────────────────────┐
│                   Nginx (反向代理)                        │
│         /api/*  →  后端 PHP 服务                          │
│         /*      →  前端静态资源                            │
└──────────────────────────┬──────────────────────────────┘
                           │
        ┌──────────────────┴──────────────────┐
        │                                      │
┌───────▼──────────┐                ┌──────────▼─────────┐
│   后端 API 服务    │                │   微信小程序 API     │
│  ThinkPHP 8       │                │  (同后端, miniapp)   │

│  ├─ Controller    │                │  ├─ AuthController  │
│  ├─ Middleware    │                │  ├─ HomeController  │
│  ├─ Service      │                │  ├─ BusinessCtrl   │
│  ├─ Validate     │                │  └─ ProfileCtrl    │
│  └─ Event        │                └──────────┬──────────┘
└───────┬──────────┘                           │
        │                                       │
┌───────▼───────────────────────────────────────▼──────────┐
│                     数据层                                 │
│  ┌──────────┐    ┌──────────┐    ┌──────────────────┐     │
│  │  MySQL   │    │  Redis   │    │  File Cache      │     │
│  │ 业务数据  │    │ (可选)   │    │  (SimpleCache)   │     │
│  └──────────┘    └──────────┘    └──────────────────┘     │
└──────────────────────────────────────────────────────────┘
```

## 2.3 核心设计原则

- **前后端分离**: 后端纯 API 输出，前端 SPA 应用
- **RBAC 多级权限**: 菜单权限 + 按钮权限 + 接口权限 + 数据权限
- **JWT 双 Token 机制**: Access Token + Refresh Token，支持无感刷新
- **软删除**: 所有核心业务表使用软删除（`delete_time` 字段）
- **操作审计**: 通过中间件 + 事件机制自动记录操作日志
- **登录安全**: 登录失败次数限制 + 账户锁定机制
- **缓存策略**: 权限数据缓存（SimpleCache），变更时主动清除
