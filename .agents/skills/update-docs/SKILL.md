# 技术文档更新

此技能是一套用于根据源代码变更来分析、更新和创建相关文档的引导式工作流，确保代码和文档保持同步。它特别为审查Pull Request时的文档完整性检查而设计。

## 工作流程

### 1. 分析代码变更

```bash
# 检查当前分支与主分支之间的差异
git diff main...HEAD --name-only

# 获取详细的变更内容
git diff main...HEAD
```

根据预定义的映射关系（参见 [references/CODE-TO-DOCS-MAPPING.md](references/CODE-TO-DOCS-MAPPING.md)），找出与变更的代码文件相对应的文档文件。

### 2. 评估文档影响

对每个变更的代码文件，评估：

- **API变更**：函数签名、参数、返回值是否改变？
- **行为变更**：功能行为是否有变化？
- **新增功能**：是否有全新的组件、函数或配置项？
- **废弃功能**：是否有功能被标记为废弃？
- **破坏性变更**：是否有不向后兼容的变更？

### 3. 更新现有文档

对于已经存在的文档，当其对应的功能或API发生变化时：

- 更新Props/参数表格
- 更新代码示例
- 添加废弃通知
- 更新描述文本
- 遵循项目固有的文档规范（参见 [references/DOC-CONVENTIONS.md](references/DOC-CONVENTIONS.md)）

### 4. 创建新文档

当添加全新功能时，使用标准模板创建符合规范的新文档：

- API参考文档模板
- 指南文档模板
- 变更日志模板

确保新文档的结构、命名和元信息（Frontmatter）都符合项目要求。

### 5. 验证文档一致性

- 检查所有代码示例是否可运行
- 检查所有链接是否有效
- 检查术语使用是否一致
- 检查文档版本是否与代码版本匹配

## 输出格式

完成文档更新后，提供：

1. **变更摘要**：列出所有更新的文档文件
2. **更新详情**：每个文件的具体变更内容
3. **待办事项**：需要人工确认或补充的内容
4. **验证结果**：文档一致性检查结果

## 使用场景

- **分析代码变更对文档的影响**：提交代码变更后，调用此技能分析哪些文档文件需要更新。通过git diff检查差异，根据预定义的映射关系找出需要更新的文档文件。
- **更新现有的文档**：当功能或API发生变化时（如组件新增了props、函数行为变更），引导更新现有文档，包括添加或修改Props表格、更新代码示例、添加废弃通知等。
- **为新功能创建脚手架文档**：添加全新功能时（如新组件、函数或配置项），快速创建符合规范的新文档，提供标准模板确保新文档的结构、命名和元信息都符合项目要求。
