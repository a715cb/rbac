# 文档编写规范

此文件定义了项目文档的风格指南和规则，确保所有文档保持一致性和高质量。

## 通用规范

### 文件命名

- 使用小写字母和连字符：`component-name.md`
- API文档使用端点路径：`api-users.md`
- 指南文档使用动词开头：`getting-started.md`、`deploying-to-vercel.md`

### Frontmatter

每个文档文件应包含以下Frontmatter：

```yaml
---
title: 文档标题
description: 简短描述（用于SEO和搜索）
sidebar_position: 1
---
```

### 标题层级

- `#` — 页面标题（仅一个）
- `##` — 主要章节
- `###` — 子章节
- `####` — 细节说明

不要跳过标题层级。

## 内容规范

### 代码示例

- 始终指定语言：` ```typescript ` 而非 ` ``` `
- 代码示例应完整可运行
- 包含必要的import语句
- 使用有意义的变量名，避免 `foo`、`bar`

### API参考

API参考文档应包含：

1. **概述**：API的简要描述
2. **签名**：函数/方法的完整类型签名
3. **参数**：参数表格（名称、类型、必填/可选、描述）
4. **返回值**：返回类型和描述
5. **示例**：至少一个使用示例
6. **注意事项**：边界情况、限制等

### 指南文档

指南文档应包含：

1. **目标**：读者将学到什么
2. **前提条件**：需要什么知识或环境
3. **步骤**：编号的操作步骤
4. **验证**：如何确认操作成功
5. **故障排除**：常见问题和解决方案

## 写作风格

- 使用主动语态
- 保持句子简洁
- 避免行话，必要时提供解释
- 使用一致的术语
- 中英文之间加空格

## 链接规范

- 优先使用相对路径链接
- 链接文本应描述目标，避免"点击这里"
- 定期检查链接有效性
