## 二、Sidebar 侧边栏组件

### 2.1 Sidebar (index.vue) - 移动/桌面切换

**核心职责**：根据 `isMobile` 在 drawer 模式与固定 sider 模式间切换。

```
isMobile === true
    └── a-drawer (placement="left", width=208px)
        ├── closable=false
        ├── class 绑定 navTheme 主题
        ├── @close → handleClose() → changeSetting('sidebarOpened', false)
        └── 内容: <Sider />
        
isMobile === false
    └── <Sider /> 直接渲染
```

**移动端自动关闭**：

```typescript
watch(route, () => {
    isMobile.value && handleClose()
})
```

**移动端路由变化 → 自动关闭抽屉**。避免导航后抽屉遮挡新页面。

### 2.2 Sider.vue - 侧边栏主体

**组成部分**：
- `Logo` 组件（应用标识）
- `BaseMenu` 动态菜单（根据路由配置渲染）

**响应折叠状态**：
- `collapsed` 属性从 `useSetting` 获取
- 折叠时仅显示图标，展开时显示图标+文字

### 2.3 侧边栏交互触发条件

| 触发条件 | 行为 | 状态影响 |
|----------|------|----------|
| 点击折叠按钮 | toggleSidebar() | sidebarOpened切换 + Header宽度联动 |
| 移动端点击遮罩 | @close事件 | sidebarOpened=false |
| 移动端路由变化 | watch(route) | 自动关闭抽屉 |
| 窗口大小变化 | resize监听 | isMobile 阈值切换 |

## 三、Header 头部组件

### 3.1 组件结构

```
a-layout-header (固定定位)
└── .ant-pro-global-header
    ├── .header-left
    │   ├── LayoutTrigger (v-show="sideOrMobile") - 折叠按钮
    │   └── SBreadcrumb (v-show条件控制) - 面包屑导航
    ├── .header-layout-menu
    │   ├── Logo (mix/top布局显示)
    │   └── BaseMenu (top:完整树 | mix+splitMenu:一级导航)
    │   └── SBreadcrumb (mix+!splitMenu:面包屑)
    └── .layout-header-action
        └── UserMenu - 用户头像/下拉/设置入口
```

### 3.2 占位Header

```html
<header :style="getDomStyle"></header>  <!-- 占位，高度=实Header高度 -->
<a-layout-header ...>  <!-- 实际Header，固定定位 -->
```

**双层Header设计**：占位元素撑开文档流，实际Header fixed定位不占用空间，防止内容被遮挡。

### 3.3 布局模式适配

| 布局 | left区域 | center区域 | 面包屑 |
|------|----------|-----------|--------|
| side | 折叠按钮 + 面包屑 | - | 显示 |
| top | 折叠按钮(移动端) | Logo + 完整菜单树 | 不显示 |
| mix+splitMenu | 折叠按钮 | Logo + 一级菜单 | 不显示 |
| mix+!splitMenu | 折叠按钮 | Logo + 面包屑 | 显示 |
| left | 折叠按钮 + 面包屑 | - | 显示 |

### 3.4 Mix布局预计算

```typescript
const visibleTopMenus = computed(() => {
    return menus.value.filter(item => isMenuVisible(item as RouteMenu))
})
```

mix布局下仅渲染 `visible=true && !hidden` 的一级菜单项。

### 3.5 头部主题切换

| 主题 | 背景 | 文字 | 当前选中 |
|------|------|------|----------|
| light | @layout-header-light-bg | 默认 | primary-color-bg |
| dark | @layout-header-dark-bg | 白色系 | primary-color-bg |

### 3.6 折叠按钮触发器

```html
<layout-trigger v-show="sideOrMobile" @click="toggle" />
```

仅当 `layout=side` 或 `isMobile=true` 时显示。点击调用 `useSetting.toggleSidebar()`。

## 四、BaseMenu 动态菜单组件

### 4.1 RouteMenus (index.tsx) - 递归菜单渲染

**组件职责**：根据路由配置动态渲染 Ant Design Menu。

**渲染决策树**：

```
renderMenu(item)
    ├── !isMenuVisible(item) → null (不渲染)
    ├── item.children && !hideChildrenInMenu
    │   └── renderSubMenu(item)
    │       └── SubMenu → children.map(renderMenu)
    └── else
        └── renderMenuItem(item)
            └── MenuItem + router-link
```

### 4.2 菜单可见性判断

```typescript
isMenuVisible(item):
    hidden===true        → false (完全隐藏)
    visible!==1         → false (业务状态)
    status!==1          → false (禁用)
    meta.hidden===true   → false (路由元信息)
    其他                → true
```

### 4.3 菜单激活逻辑

```
activateMenu()
    ├── 从 route.matched 获取匹配路由链
    ├── 隐藏路由处理：hidden路由弹出最后一级
    ├── selectedKeys = [匹配的最后一个路由path]
    ├── 支持 meta.active_key 自定义高亮
    ├── inline模式 → openKeys = 所有祖先路由path
    └── 折叠时缓存 openKeys → 展开时恢复
```

### 4.4 折叠状态联动

```
collapsed变化
    ├── true → cachedOpenKeys=openKeys, openKeys=[]
    └── false → openKeys=cachedOpenKeys
```

折叠时关闭所有展开子菜单，展开时恢复之前的展开状态。

### 4.5 SubMenu 展开逻辑

```
onOpenChange(openKeys)
    ├── horizontal模式 → 直接使用新keys
    └── 其他模式
        ├── 找到最新打开的key
        ├── 是顶级菜单key → 只保留它（手风琴）
        └── 不是顶级 → 保留所有
```

## 五、TagsView 多标签页组件

### 5.1 组件结构

```
.multi-tab-stuff (占位元素，fixedMultiTabs时显示)
.tabs
└── a-tabs (type="editable-card", hide-add)
    ├── a-tab-pane (遍历list)
    │   └── #tab: SIcon + title
    └── #rightExtra: a-dropdown
        └── a-menu
            ├── 关闭其他 → closeOther()
            ├── 关闭当前 → closeCurrent()
            └── 关闭全部 → closeAll()
```

### 5.2 标签页操作

| 操作 | 触发 | 行为 |
|------|------|------|
| 点击标签 | onChange(path) | 查找路由信息 → navigation({path, query}) |
| 点击关闭 | onEdit('remove') | close(path) |
| 关闭其他 | dropdown | closeOther() |
| 关闭当前 | dropdown | closeCurrent() |
| 关闭全部 | dropdown | closeAll() |

### 5.3 标签页特性

- **affix标签**：不可关闭（仪表盘默认affix）
- **图标**：显示菜单icon，无icon时使用默认图标
- **query保留**：跳转时保留原始query参数
- **位置**：支持固定/非固定两种模式

### 5.4 固定标签页

```
fixedMultiTabs=true
    └── .multi-tab-stuff (占位) + .multi-tab-fixed (固定定位)
```

固定标签页始终显示在页面顶部，内容区自动避让。

## 六、SettingDrawer 设置抽屉组件

### 6.1 组件结构

```
.setting-container
└── a-drawer (width=310px, placement="right")
    ├── 自定义关闭按钮 (header模式)
    ├── .setting-drawer-index-content
    │   ├── 整体风格设置 (light/dark/realDark)
    │   ├── 主题色设置 (8种预设颜色)
    │   ├── 导航模式 (side/top/mix/left)
    │   └── 其他设置
    │       ├── 路由动画 + 动画类型
    │       ├── 分割菜单
    │       ├── 面包屑
    │       ├── 标签页 + 固定标签页 + 标签页类型
    │       ├── 页面加载条
    │       ├── 圆角
    │       ├── 底部栏
    │       └── 主题设置位置
    └── #handle: 固定把手 (fixed模式)
```

### 6.2 两种入口模式

| 模式 | 入口 | 关闭按钮 | 把手 |
|------|------|----------|------|
| header | 头部UserMenu → inject('openSetting') | 自定义 × 按钮 | 隐藏 |
| fixed | 页面右侧把手 | 无(把手切换) | 齿轮/关闭图标 |

### 6.3 抽屉滑入动画

```css
:global(.setting-drawer .ant-drawer-content-wrapper) {
    transform: translate(100%);  /* 初始隐藏在右侧 */
}
:global(.setting-drawer-show .ant-drawer-content-wrapper) {
    transform: translate(0);      /* 展开在视图内 */
}
```

通过 `rootClassName` 动态添加/移除 `setting-drawer-show` 类触发动画。

### 6.4 主题切换交互

```
用户点击主题风格图标
    ↓
setTheme(newTheme) → changeTheme(newTheme)
    ↓
├── 更新 theme 状态 (useSetting)
├── 更新 data-theme HTML属性
├── 触发 App.vue themeSetting 重新计算
└── Ant Design 全局主题令牌更新
```

### 6.5 主题色切换

```
用户点击颜色色块
    ↓
changeColor(color) → changeSetting('primaryColor', color)
    ↓
├── 更新 primaryColor 状态
├── App.vue themeSetting.token.colorPrimary 更新
└── 所有 Ant Design 组件的主题色变更
```

### 6.6 布局切换联动

```
layout 变化时自动调整关联设置：

side → splitMenu=false, showBreadcrumb=true
top/mix → splitMenu=true
mix/top → showBreadcrumb=false
left → showBreadcrumb=true
```

### 6.7 设置项依赖关系

| 设置项 | 依赖条件 |
|--------|----------|
| 动画类型 | openAnimation=true 才可用 |
| 分割菜单 | 仅 side 布局可切换 (!['top','mix','left'].includes(layout)) |
| 面包屑 | 仅 side 布局可切换 |
| 标签页类型 | showMultiTabs=true 才有意义 |

## 八、异常处理汇总

| 组件 | 异常 | 处理 |
|------|------|------|
| Sidebar | 移动端路由变化时抽屉未关闭 | watch(route) → handleClose() |
| Header | 菜单数据为空 | visibleTopMenus为空 → 不渲染菜单 |
| BaseMenu | 菜单项无可见子项 | renderMenu返回null |
| TagsView | 关闭最后一个标签 | affix标签不可关闭 |
| SettingDrawer | 布局切换时设置冲突 | watch(layout) 自动修正关联设置 |
| SettingDrawer | toggle调用时ref未就绪 | nextTick保护 |
