# RBAC 权限管理系统 - 登录页交互分析报告

**文档版本**：1.0 | **生成日期**：2026-05-12 | **页面路径**：`src/pages/login/index.vue`

---

## 一、页面概述

### 1.1 页面定位

登录页是系统的入口页面，采用左右分栏布局：左侧为品牌展示区（渐变背景 + 功能特性介绍），右侧为登录表单区。页面不依赖 DefaultLayout，独立渲染。

### 1.2 页面功能模块划分

| 模块 | 位置 | 功能 |
|------|------|------|
| 品牌展示区 | 左侧 50% | 展示系统名称、功能特性列表 |
| 登录表单区 | 右侧 50% | 用户名、密码、验证码输入 + 登录提交 |
| 记住密码区 | 表单底部 | 记住用户名到 localStorage |
| 辅助链接 | 表单底部 | 忘记密码、立即注册（占位功能） |

### 1.3 依赖组件

| 组件 | 用途 |
|------|------|
| `Captcha` | 图形验证码生成与校验 |
| `useUserStore` | 登录成功后存储 Token 和用户信息 |
| `login` API | 向后端发送登录请求 |
| `StorageManager` | 记住密码功能的数据持久化 |

## 二、用户操作流程

### 2.1 完整登录流程图

```
页面加载 onMounted
    │
    ├── loadRememberedUsername()
    │   └── 从 localStorage 读取上次记住的用户名
    │       └── 有数据 → 填充用户名 + 勾选"记住密码"
    │
    ▼
用户输入用户名（3-20字符，字母开头）
    │  └── blur 触发格式校验
    ▼
用户输入密码（最少6位）
    │  └── blur 触格长度校验
    │  └── 可点击眼睛图标切换密码可见性
    ▼
用户输入验证码（4位数字）
    │  └── 可点击刷新按钮重新获取验证码
    │
    ▼
用户点击"登 录"按钮
    │
    ├── [1] 验证码前端校验
    │   └── captchaRef.value.validate() → 失败 → 提示 + 刷新验证码 → 终止
    │
    ├── [2] 验证码长度校验
    │   └── formState.captcha.length !== 4 → 提示 → 终止
    │
    ├── [3] 防重复提交
    │   └── loading=true + isSubmitting=true → 按钮显示loading + disabled
    │
    ├── [4] 调用 login() API
    │   ├── 成功 → 存储Token → 存储用户信息 → 记住用户名 → 跳转目标页
    │   └── 失败 → 错误分类提示 → 刷新验证码
    │
    └── [5] finally → loading=false + isSubmitting=false
```

### 2.2 登录成功后续流程

```
登录成功
  │
  ├── 提取响应数据：access_token, refresh_token, expires_in, user_info
  │
  ├── userStore.setTokenWithExpiry(access_token, refresh_token, expires_in)
  │   └── 存储到 sessionStorage + localStorage (refreshToken)
  │
  ├── userStore.setUserInfo(user_info)
  │   └── 存储用户信息到 sessionStorage
  │
  ├── 记住密码逻辑
  │   ├── remember=true → 存储用户名到 localStorage
  │   └── remember=false → 清除 localStorage 中存储的用户名
  │
  └── router.push(redirect || '/dashboard')
```

## 三、交互触发条件与状态反馈

### 3.1 表单输入交互

| 交互元素 | 触发条件 | 即时反馈 | 验证反馈 |
|----------|----------|----------|----------|
| 用户名输入框 | 用户输入 | 无 | blur时校验：必填/长度3-20 |
| 密码输入框 | 用户输入 | 密码遮蔽/可见切换 | blur时校验：必填/长度≥6 |
| 验证码输入框 | 用户输入 | 无 | blur时校验：必填/长度=4 |
| 记住密码复选框 | 点击切换 | 选中/取消视觉变化 | 无 |
| 登录按钮 | 点击 | loading动画 | disabled状态 |

### 3.2 表单验证规则

| 字段 | 规则 | 触发时机 |
|------|------|----------|
| username | required + min:3 + max:20 | blur |
| password | required + min:6 | blur |
| captcha | required + len:4 | blur |

### 3.3 表单验证失败处理

| 触发方式 | 处理方式 |
|----------|----------|
| `@finish-failed` | 显示"请检查表单填写是否正确"提示 |
| 验证码校验失败 | 显示"验证码不正确，请重新输入" + 自动刷新验证码 |
| 验证码长度不足 | 显示"请输入 4 位验证码" |

### 3.4 登录失败错误分类

| 错误关键词 | 用户提示 |
|------------|----------|
| "用户名或密码错误" | 用户名或密码错误，请重新输入 |
| "账户已被禁用" | 账户已被禁用，请联系管理员 |
| "锁定" | 原始错误消息 |
| "网络"/"超时"/"连接" | 网络连接异常，请检查网络后重试 |
| "服务器"/"500" | 服务器繁忙，请稍后重试 |
| 其他 | 原始错误消息 |

## 六、异常处理逻辑

| 异常场景 | 处理机制 |
|----------|----------|
| 网络断开 | catch 捕获 → 错误分类 → "网络连接异常"提示 |
| 登录接口超时 | axios timeout → "请求超时"提示 |
| Token 存储失败 | setTokenWithExpiry 内部 try-catch |
| 记住的用户名读取失败 | loadRememberedUsername 静默忽略 |
| 验证码组件未挂载 | captchaRef.value 判空 |

## 七、与其他页面/组件的交互关联

| 关联组件/页面 | 交互方式 | 数据流向 |
|--------------|----------|----------|
| `useUserStore` | 直接调用 | Token/用户信息 → Store → sessionStorage |
| `TokenManager` | 间接 (通过 Store) | Token → localStorage |
| Dashboard 页 | router.push | 登录成功后跳转 |
| 任意目标页 | router.push(redirect) | 从 URL query 参数获取 |
| `Captcha` 组件 | ref 调用 validate/refresh | 父组件控制子组件 |

## 八、用户体验细节

| 细节 | 说明 |
|------|------|
| 输入框前缀图标 | 用户名前显示 UserOutlined，密码前显示 LockOutlined |
| 浏览器自动填充兼容 | 多处 `:-webkit-autofill` 样式覆盖确保透明背景 |
| 无障碍标签 | 表单标签使用 `sr-only` 方式（绝对定位隐藏但屏幕阅读器可读） |
| 表单标签 id 关联 | `html-for` 属性关联 label 和 input |
| autocomplete 控制 | 新增模式下 `autocomplete="new-password"` 防止自动填充 |
