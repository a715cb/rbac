# RBAC 权限管理系统 - 角色管理页交互分析报告

**文档版本**：1.0 | **生成日期**：2026-05-12 | **页面路径**：`src/pages/system/role/index.vue`

## 一、页面概述

### 1.1 页面定位

角色管理页是 RBAC 权限体系的核心管理页面，负责角色的增删改查、状态切换、权限分配以及数据范围设置。权限分配和数据范围是角色特有的高级功能。

### 1.2 页面功能模块划分

| 模块 | 位置 | 功能 |
|------|------|------|
| 搜索表单区 | 页面顶部 | 关键词 + 状态（字典驱动）筛选 |
| 表格工具栏 | 表格上方 | 新增按钮 + 列设置 |
| 数据表格 | 页面主体 | 分页角色列表 |
| 角色表单弹窗 | 模态层 | 新增/编辑角色 |
| 权限分配弹窗 | 模态层 | 为角色分配菜单/按钮权限 |
| 数据范围弹窗 | 模态层 | 设置角色数据访问边界 |

### 1.3 依赖组件

| 组件 | 用途 |
|------|------|
| `RoleFormModal` | 角色新增/编辑表单弹窗 |
| `RolePermissionModal` | 权限分配弹窗（树形菜单选择） |
| `RoleDataScopeModal` | 数据范围设置弹窗 |
| `TableSetting` | 表格列设置 |
| `DictSelect` | 字典选择器（状态筛选） |
| `usePageTable` | 表格设置状态管理 |
| `useDict` | 字典数据缓存 |

## 二、用户操作流程

### 2.1 页面初始化

```
onMounted → fetchData()
    ├── loading = true
    ├── getRoleList({ page, limit, keyword, status })
    │   ├── 成功 → tableData + pagination.total 更新
    │   └── 失败 → console.error(DEV)
    └── loading = false
```

### 2.2 新增角色

```
用户点击"新增" → currentRecord=null → modalVisible=true
    │
    ▼
RoleFormModal 打开 (新增模式)
    ├── 填写角色名/标识/排序/状态/备注
    ├── 点击确定 → createRole()
    └── success → emit('success') → handleSearch()
```

### 2.3 编辑角色

```
用户点击行"编辑" → currentRecord=record → modalVisible=true
    │
    ▼
RoleFormModal 打开 (编辑模式，预填数据)
    └── 更新成功 → 刷新列表
```

### 2.4 权限分配

```
用户点击行"权限"按钮
    │
    ▼
handlePermissions(record)
    ├── currentRecord = record
    └── permissionModalVisible = true
        │
        ▼
RolePermissionModal 打开
    ├── 展示菜单树 (含按钮级权限)
    ├── 用户勾选/取消菜单节点
    ├── 支持父子联动选择
    └── 保存 → emit('success') → handleSearch()
```

### 2.5 数据范围设置

```
用户点击行"数据"按钮
    │
    ▼
handleDataScope(record)
    ├── currentRecord = record
    └── dataScopeModalVisible = true
        │
        ▼
RoleDataScopeModal 打开
    ├── 选择数据范围类型: 全部/本部门/本部门及下级/仅本人/自定义
    ├── 自定义模式 → 选择具体部门
    └── 保存 → emit('success') → handleSearch()
```

### 2.6 删除与状态切换

| 操作 | 触发 | 确认方式 | 成功后 |
|------|------|----------|--------|
| 删除角色 | 点击"删除" | Popconfirm | message.success + fetchData() |
| 状态切换 | Switch点击 | 直接执行 | message.success + fetchData() |

## 三、交互触发条件与状态反馈

### 3.1 表格列自定义渲染

| 列名 | 渲染方式 | 颜色映射 |
|------|----------|----------|
| 数据范围 | getDictLabel + Tag | 1=蓝, 2=绿, 3=橙, 4=紫, 5=红 |
| 用户数 | Badge数字徽标 | 蓝色 #108ee9 |
| 状态 | Switch开关 | 点击即时切换 |
| 操作 | 编辑/权限/数据/删除 | 删除需Popconfirm |

### 3.2 数据范围颜色映射

| 范围值 | 含义 | Tag颜色 |
|--------|------|---------|
| 1 | 全部数据 | blue |
| 2 | 本部门 | green |
| 3 | 本部门及下级 | orange |
| 4 | 仅本人 | purple |
| 5 | 自定义 | red |

### 3.3 搜索筛选

| 筛选方式 | 组件 | 数据源 |
|----------|------|--------|
| 关键词 | a-input | 角色名称/标识模糊匹配 |
| 状态 | DictSelect(dict-code="role_status") | 字典管理系统 |

## 六、异常处理逻辑

| 异常 | 处理 |
|------|------|
| 列表加载失败 | console.error(DEV) + 拦截器显示 |
| 删除失败 | console.error(DEV) |
| 状态切换失败 | console.error(DEV) + fetchData()回滚 |
| 权限分配失败 | 弹窗不关闭，用户可重试 |
