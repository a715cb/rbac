**文档版本**：1.0 | **生成日期**：2026-05-12 | **页面路径**：`src/pages/system/dict/index.vue`

## 一、页面概述

### 1.1 页面定位

字典管理页采用master-detail左右分栏布局：左侧为字典类型列表（支持搜索和快速操作），右侧为选中类型的字典数据表格。提供字典类型和字典数据的完整CRUD功能。

### 1.2 页面功能模块划分

| 模块 | 位置 | 功能 |
|------|------|------|
| 类型搜索 | 左侧顶部 | a-input-search 搜索字典名称 |
| 类型列表 | 左侧主体 | 字典类型列表 + 快速操作 |
| 类型操作栏 | 左侧底部 | "添加字典"按钮 |
| 数据表格 | 右侧 | 选中类型的字典数据表格 |
| 空状态 | 右侧 | 未选择类型时的空提示 |
| 类型表单弹窗 | 模态层 | DictTypeModal 新增/编辑 |
| 数据表单弹窗 | 模态层 | DictDataModal 新增/编辑 |

### 1.3 依赖组件

| 组件 | 用途 |
|------|------|
| `DictTypeModal` | 字典类型表单弹窗 |
| `DictDataModal` | 字典数据表单弹窗 |
| `TableSetting` | 表格列设置 |
| `useTableSetting` | 表格设置状态 |

## 二、用户操作流程

### 2.1 页面初始化

```
onMounted → fetchTypeList(true)
    ├── typeLoading = true
    ├── getDictTypeList() → typeList更新
    ├── isInit=true且列表非空 → currentType=第一项
    └── typeLoading = false

watch(currentType) → fetchDataList()
    └── 获取选中类型的字典数据
```

### 2.2 字典类型操作

| 操作 | 触发 | 效果 |
|------|------|------|
| 搜索类型 | 输入关键词 | computed filteredTypeList实时过滤 |
| 选择类型 | 点击列表项 | currentType=该类型 + 右侧加载数据 |
| 新增类型 | 点击"添加字典" | DictTypeModal打开(新增模式) |
| 编辑类型 | 点击编辑图标(@click.stop) | DictTypeModal打开(编辑模式) |
| 启用/禁用 | 点击对应图标 | Modal.confirm确认 → changeDictTypeStatus |
| 删除类型 | 点击删除图标 | Modal.confirm确认(提示级联删除) |

### 2.3 字典数据操作

| 操作 | 触发 | 效果 |
|------|------|------|
| 新增数据 | 点击"添加" | DictDataModal打开(新增模式,传入dictTypeId) |
| 编辑数据 | 点击行"编辑" | DictDataModal打开(编辑模式) |
| 删除数据 | 点击行"删除"(Popconfirm) | deleteDictData → fetchDataList() |
| 状态切换 | Switch点击 | changeDictDataStatus → fetchDataList() |

### 2.4 类型列表hover交互

列表项hover时：显示操作图标（编辑/启用禁用/删除），隐藏类型编码(code)。通过CSS控制。

```css
.custom-dict-node:hover {
    .action { display: block; }
    .type { display: none; }
}
```

## 三、交互触发条件与状态反馈

### 3.1 选中状态

```css
.active {
    background-color: var(--ant-color-primary-bg, #e6f4ff);
```

```html

```

### 3.3 表格列

| 列 | 特性 |
|----|------|
| ID | 固定宽度80 |
| 字典标签/键值 | 纯展示 |

使用 `useTableSetting` + `createTableSettingContext` 模式管理列配置和表格密度。

## 六、异常处理

| 异常 | 处理 |
|------|------|
| 类型列表加载失败 | console.error(DEV) |
| 数据列表加载失败 | console.error(DEV) |
| 删除类型后选中类型变化 | 保持或回退到第一个 |
| 切换状态失败 | fetchTypeList()/fetchDataList()回滚 |

### 6.1 类型列表刷新后的状态保持

```typescript
if (currentType.value) {
    const updated = typeList.value.find(item => item.id === currentType.value!.id)
    if (updated) currentType.value = updated  // 保持选中
    else if (typeList.value.length > 0) currentType.value = typeList.value[0]
    else currentType.value = null
}
```
