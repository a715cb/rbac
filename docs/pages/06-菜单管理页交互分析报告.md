# RBAC 权限管理系统 - 菜单管理页交互分析报告

**文档版本**：1.0 | **生成日期**：2026-05-12 | **页面路径**：`src/pages/system/menu/index.vue`

## 一、页面概述

### 1.1 页面定位

菜单管理页以递归树形表格展示系统的菜单层级结构（目录→菜单→按钮），支持节点的增删改查、全局展开/折叠、按名称搜索高亮，并在操作后自动刷新前端菜单缓存。

### 1.2 页面功能模块划分

| 模块 | 位置 | 功能 |
|------|------|------|
| 表格工具栏 | 表格上方 | 新增、展开全部、折叠全部按钮 + 列设置 |
| 树形表格 | 页面主体 | 树形结构展示菜单层级数据（分页禁用） |
| 搜索筛选 | 表格列头 | 名称列的筛选下拉框（搜索+高亮） |
| 表单弹窗 | 模态层 | 新增/编辑菜单（含父级ID传递） |

### 1.3 依赖组件与工具

| 依赖 | 用途 |
|------|------|
| `MenuFormModal` | 菜单表单弹窗（支持新增/编辑/添加子菜单） |
| `SIcon` | 菜单图标渲染 |
| `TableSetting` | 表格列设置 |
| `usePageTable` | 表格设置状态管理 |
| `useTreeSearch` | 树形搜索 + 文本高亮 + 展开匹配节点 |
| `useDebounceFn` | 防抖搜索（300ms） |
| `StorageManager` | 菜单缓存清理 |
| `useUserStore` | 重置 dynamicRoutesAdded 标识 |

## 二、用户操作流程

### 2.1 页面初始化

```
onMounted → fetchData()
    ├── loading = true
    ├── getMenuList() → tableData (id转为Number)
    ├── expandedRowKeys = [] (初始全部折叠)
    └── loading = false
```

### 2.2 新增顶级菜单

```
点击"新增" → handleAdd()
    ├── currentRecord = null
    ├── parentId = undefined
    └── modalVisible = true
```

### 2.3 新增子菜单

```
点击行"添加" → handleAddChild(record)
    ├── currentRecord = null
    ├── parentId = record.id (传递父级ID)
    └── modalVisible = true
```

### 2.4 编辑菜单

```
点击行"编辑" → handleEdit(record)
    ├── currentRecord = record
    ├── parentId = undefined
    └── modalVisible = true
```

### 2.5 删除菜单

```
点击行"删除" (Popconfirm确认)
    │
    ▼
handleDelete(record)
    ├── deleteMenu(record.id)
    ├── fetchData() 刷新列表
    └── refreshMenuCache() 清除前端菜单缓存
```

### 2.6 展开/折叠控制

```
点击"展开" → expandAll()
    └── collectExpandableIds(tableData) → expandedRowKeys = 所有可展开节点ID

点击"折叠" → collapseAll()
    └── expandedRowKeys = []

点击展开图标 → handleExpand(expanded, record)
    ├── 展开 → expandedRowKeys 添加 record.id
    └── 折叠 → expandedRowKeys 过滤掉 record.id
```

### 2.7 搜索高亮

```
用户点击菜单名称列筛选图标 → 下拉搜索框打开
    ├── 自动聚焦搜索输入框 (setTimeout 100ms)
    │
    ├── 输入搜索词 → 防抖300ms → debouncedDoSearch(keyword)
    │   └── doSearch(keyword, tableData)
    │       ├── 匹配 name 字段
    │       ├── 展开包含匹配节点的祖先
    │       └── 标记匹配文本用于高亮
    │
    ├── 点击"搜索"按钮 → handleFilterSearch
    │   └── confirm() + debouncedDoSearch(keyword)
    │
    └── 点击"重置"按钮 → handleFilterReset
        └── clearFilters() + resetSearch() + confirm()
```

### 2.8 菜单缓存刷新

```typescript
const refreshMenuCache = () => {
  StorageManager.removeItem('session', AppConfig.menusKey)
  userStore.dynamicRoutesAdded = false
}
```

**触发时机**：删除菜单成功、菜单表单操作成功后调用。

## 三、交互触发条件与状态反馈

### 3.1 表格列自定义渲染

| 列名 | 渲染方式 | 交互特性 |
|------|----------|----------|
| 菜单名称 | 搜索文本高亮 + 自定义筛选下拉 | innerHTML渲染高亮文本 |
| 图标 | SIcon组件渲染，无图标显示"-" | 纯展示 |
| 菜单类型 | 彩色Tag | 1蓝(目录)/2绿(菜单)/3橙(按钮) |
| 状态 | 文本颜色区分 | 正常黑色/禁用红色 |
| 排序 | 数字居中展示 | 纯展示 |
| 操作 | 添加/编辑/删除按钮 | 删除需Popconfirm确认 |

### 3.2 树形展开图标

| 节点类型 | 展示方式 |
|----------|----------|
| 可展开节点 | CaretRightOutlined 右箭头，展开时旋转90度 |
| 叶子节点 | 占位符(20px宽度)保持缩进对齐 |

**展开图标动画**：
```css
.tree-expand-icon .anticon {
    transition: transform 0.2s ease-in-out;
}
.tree-expand-icon.expanded .anticon {
    transform: rotate(90deg);
}
```

### 3.3 搜索高亮机制

| 交互阶段 | 行为 |
|----------|------|
| 输入搜索词 | 防抖300ms → 匹配数据 |
| 匹配过程 | 递归遍历 tableData → 标记匹配节点 |
| 高亮渲染 | highlightText() 包裹匹配文本（带高亮样式） |
| 展开联动 | 自动展开包含匹配节点的祖先路径 |

## 六、异常处理逻辑

| 异常 | 处理 |
|------|------|
| 列表加载失败 | console.error(DEV) |
| 删除失败 | console.error(DEV) + 拦截器显示错误 |
| 搜索匹配为空 | 无结果，不展开任何节点 |
| 菜单缓存清理失败 | StorageManager.removeItem 静默处理 |

## 七、与其他页面/组件的交互关联

| 关联 | 方式 | 影响 |
|------|------|------|
| MenuFormModal | v-model:visible + @success | 操作成功后刷新列表和缓存 |

| getMenuList/deleteMenu | API调用 | 数据CRUD |
